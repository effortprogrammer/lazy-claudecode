# Start Work Skill

## Session Initialization Protocol

### 1. Orientation
- Read the project's CLAUDE.md, AGENTS.md, or README.md
- Understand the project structure (key directories, entry points)
- Check for any active plans or work-in-progress

### 2. Context Gathering
- `git status` — Check current branch and uncommitted changes
- `git log --oneline -5` — Review recent history
- Check for open TODOs or issues in the current context

### 3. Goal Setting
- Clarify what needs to be done in this session
- Break down into concrete, verifiable tasks
- Prioritize by impact and dependency order

### 4. Environment Check
- Verify build/test commands work
- Check for any failing tests (baseline)
- Ensure tooling is available (LSP, linters, etc.)

### 5. Begin Work
- Start with the highest-priority task
- Work systematically through the plan
- Verify each step before moving to the next
