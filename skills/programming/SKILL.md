# Programming Skill

## Core Principles
- Write clean, readable, self-documenting code
- Follow the language's idiomatic patterns and conventions
- Prefer composition over inheritance
- Keep functions small and focused (single responsibility)
- Use meaningful names for variables, functions, and types
- Handle errors explicitly — never swallow errors silently
- Write code for the next reader, not the compiler

## Process
1. **Understand** the requirements and constraints
2. **Read** existing code to understand patterns and conventions
3. **Plan** the implementation approach
4. **Implement** incrementally with verification at each step
5. **Test** that the code works correctly
6. **Review** for edge cases, error handling, and code quality

## Anti-Patterns to Avoid
- Premature optimization
- Deep nesting (use early returns)
- Magic numbers and strings (use constants)
- Unnecessary comments that restate the code
- Overly clever code that's hard to understand
- Copy-paste without understanding
- Ignoring compiler/linter warnings
